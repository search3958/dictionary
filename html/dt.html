<dt>

Definition Term（定義用語）要素。定義リスト（<dl>）内で、定義する用語や名前を指定します。

info

概要

<dt> 要素（定義用語要素）は、定義リスト <dl> の子要素として使用され、リスト内の用語や名前を示します。
必ずその直後に、対応する定義や説明を示す <dd> 要素が続く必要があります（ただし、複数の <dt> が一つの <dd> に対応することも可能です）。
例：用語集、メタデータ（名前と値のペア）、FAQ（質問と回答）などに適しています。

play_circle

デモ (静的表示)

HTML (Source)
<dl> <dt>HTML</dt> <dd>Webページの構造を記述する言語です。</dd> <dt>CSS</dt> <dd>Webページの見た目を装飾する言語です。</dd> </dl>
Preview (Result)
HTML
Webページの構造を記述する言語です。
CSS
Webページの見た目を装飾する言語です。
list

属性 (Attributes)

属性名 説明
（固有属性なし） <dt> 要素には固有の属性はありません。
グローバル属性 全てのHTML要素で使用できる属性（class, id, style, title, data-*など）のみ使用可能です。
code

実用例

1. 基本的な定義リスト

<dl> <dt>作成者</dt> <dd>山田 太郎</dd> <dt>公開日</dt> <dd>2025年12月14日</dd> </dl>

2. 複数の用語に対する一つの定義

<dl> <dt>PC</dt> <dt>Personal Computer</dt> <dd>個人用のコンピュータの総称。</dd> </dl>

3. FAQ（質問と回答）としての利用

<dl> <dt>Q: 注文のキャンセルはできますか？</dt> <dd>A: 出荷前であれば可能です。マイページから手続きを行ってください。</dd> </dl>
lightbulb

知っておくべき重要事項

assignment
親要素は <dl> のみ
<dt> 要素は、定義リスト <dl> の直下でのみ使用できます。他のブロック要素（例: <div> や <p>）の中に直接配置することはできません。
accessibility_new
セマンティクス（意味付け）の重要性
<dt> は、単に太字のテキストとして使うべきではありません。これは「用語」「名前」というセマンティクスを持っています。リスト形式にしたいだけであれば、<ul>/<li> や <ol>/<li> を検討してください。
